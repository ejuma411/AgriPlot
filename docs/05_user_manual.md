# User Manual

## 1. Introduction
AgriPlot Connect helps users list, verify, discover, and transact agricultural land. Different users see different tools depending on their role.

## 2. User Roles
- Buyer: browse plots, save plots, join waitlists, create payments, follow transaction progress.
- Landowner: publish plots and monitor verification and transaction activity.
- Agent: manage plot listings and buyer engagement.
- Verification admin: assign and supervise verification tasks.
- Extension officer: complete field agronomy reviews.
- Land surveyor: complete survey inspections and upload survey evidence.

## 3. Getting Started
1. Open the platform homepage.
2. Sign in through the login page.
3. Complete any required profile or verification steps.
4. Use the dashboard appropriate to your role.

## 4. Browsing the Marketplace
1. Use the homepage search bar to describe the land you want.
2. Refine results using county, sub-county, price, soil type, size intent, and other filters.
3. Open a plot card to view full details.
4. Save a plot if you want to revisit it later.

## 5. Creating and Managing Listings
1. Go to the listing dashboard.
2. Add a plot with title, location, size, pricing, and supporting details.
3. Upload required supporting documents.
4. Submit the plot into the verification process.

## 6. Verification Workflow
### For Verification Admins
1. Open the verification dashboard.
2. Review incoming queue items.
3. Assign tasks to extension officers or surveyors.
4. Review submitted evidence and continue the plot through approval or follow-up.

### For Land Surveyors
1. Open the surveyor dashboard.
2. Open the assigned inspection task.
3. Enter visit details and required survey findings.
4. Upload the mutation form and any plot images.
5. Submit the report.

### For Extension Officers
1. Open the extension dashboard.
2. Open the assigned field review task.
3. Complete agronomic and land-use findings.
4. Submit the report.

## 7. Creating a Payment
1. Open a plot or payment workflow page.
2. Start a payment request for the relevant stage.
3. Confirm the linked plot and transaction type.
4. Enter the M-Pesa number or selected payment details.
5. Submit the request.

Important:
- Transactional payments such as commitment, reservation, agreement, escrow, stamp duty, and completion balance require a linked plot.

## 8. If Provider Confirmation Is Delayed
If Safaricom does not confirm the STK push immediately:

- the payment request will still be created;
- the payment will remain pending;
- the detail page can be used for follow-up and later confirmation.

This prevents the transaction record from being lost when the provider is slow.

## 9. Following Transaction Progress
1. Open the payment detail page.
2. Review current status, payment history, and closing steps.
3. Complete the next required legal or operational step.
4. Monitor status until the workflow reaches completion.

## 10. Notifications
Users may receive:
- in-app notifications;
- email notifications;
- SMS notifications when enabled and available.

Notification delivery problems do not cancel the underlying business action.

## 11. Reports and Admin Tools
Authorized staff can open reporting and audit tools to review:
- escrow and transaction summaries;
- executive reports;
- audit logs;
- verification performance and operational dashboards.

## 12. Troubleshooting
### A payment form will not submit
- Confirm a plot is selected.
- Confirm required fields such as phone number are present.
- Check validation messages on the page.

### A surveyor upload does not appear
- Reopen the related report and confirm the upload was submitted successfully.
- Confirm the image is a supported file type.

### A verification link no longer works under `/staff/...`
- Use the canonical root verification routes instead.

### Filters on the homepage are not updating
- Submit the search form normally.
- Use the reset button to clear active filters.

## 13. Support Guidance
If a workflow is blocked by missing provider confirmation, legal evidence, or permissions, contact the platform support or administrative team with the payment reference, plot title, and the step where the workflow stopped.
